# slacker αναφορά blacklist (Ελληνικά)

Πώς να προσθέτεις πακέτα και patterns στη blacklist του slacker, είτε με την εντολή
`frozen` είτε επεξεργάζοντας το αρχείο με το χέρι. Αυτός είναι ένας πλήρης κατάλογος
των μορφών κανόνων και των εντολών που τους δημιουργούν.

## Τι κάνει η blacklist

Ένας κανόνας blacklist ταιριάζει πακέτα. Το αποτέλεσμα εξαρτάται από το αν ένα
πακέτο που ταιριάζει είναι εγκατεστημένο:

- Εγκατεστημένο και ταιριάζει: το πακέτο **παγώνει**. Δεν εγκαθίσταται,
  αναβαθμίζεται, επανεγκαθίσταται ή αφαιρείται ποτέ από το slacker, και δεν
  εμφανίζεται ποτέ στο `clean-system`.
- Μη εγκατεστημένο και ταιριάζει: το πακέτο **κρύβεται** από το `install-new`, από
  τις αναβαθμίσεις και από το `check-updates`. Εξακολουθεί να εμφανίζεται από το
  `search` και το `info`, σημειωμένο `[blacklisted]`.

## Πάγωμα πακέτου vs. καραντίνα αποθετηρίου

Η blacklist παγώνει/κρύβει μεμονωμένα **πακέτα**. Η επέμβαση σε ολόκληρο
**αποθετήριο** είναι ξεχωριστός μηχανισμός — *quarantine*:

- το slacker βάζει αυτόματα σε quarantine ένα repo που αποτυγχάνει στο vetting
  (απρόσιτο, ή σερβίρει κακοσχηματισμένα/εχθρικά metadata)· ένα repo σε quarantine
  δεν παρέχει κανένα πακέτο.
- `slacker distrust-repo NAME` — βάλε ένα repo σε quarantine εσύ ο ίδιος·
- `slacker vet-repo NAME` — επανέλεγξέ το on demand (αίρει την quarantine αν τώρα
  περάσει)·
- `slacker trust-repo NAME` — άρε μια quarantine που κρίνεις false positive.

Πρακτικός κανόνας: χρησιμοποίησε τη **blacklist** για μεμονωμένα πακέτα, την
**quarantine** για ολόκληρα αποθετήρια. (Για να κρατήσεις τα πακέτα ενός repo αλλά
να εξακολουθεί να τρέχει το `clean-system`, δες το flag `immutable` παρακάτω, όχι
την quarantine.)

## Διατήρηση πακέτου εκτός clean-system

Το πάγωμα ενός πακέτου με τη blacklist είναι **ένας από τους τρεις** τρόπους να το
κρατήσεις εκτός `clean-system`, που αφαιρεί πακέτα που δεν είναι πια στο επίσημο
baseline (το επίσημο `PACKAGES.TXT` συν οποιοδήποτε repo μαρκαρισμένο `immutable`):

- ένας κανόνας **blacklist** — ανά πακέτο (αυτό το αρχείο)·
- το **build tag** του στο `IGNORE_TAGS` (στο `slacker.conf`), π.χ. `_SBo cf alien`·
- το repo του μαρκαρισμένο **`immutable`** στο `repos`, που κρατά κάθε πακέτο που
  αποδίδεται σε αυτό το repo.

Για ένα ολόκληρο Slackware subtree repo (`extra/`, `testing/`, `patches/`)
προτίμησε να μαρκάρεις το repo `immutable` αντί να βάζεις στη blacklist κάθε πακέτο
με το χέρι.

## Σύνταξη κανόνων

Κάθε καταχώρηση είναι ένας κανόνας:

```
[@repo] PATTERN
```

- Ένα `PATTERN` που τελειώνει σε `/` είναι κανόνας **σειράς** (π.χ. `kde/`)·
  ταιριάζει μια σειρά (series) του Slackware.
- Οποιοδήποτε άλλο `PATTERN` είναι μια **κανονική έκφραση** χωρίς άγκυρα, που
  ταιριάζεται έναντι του πλήρους id πακέτου `name-version-arch-build` (στυλ
  slackpkg).
- Το προαιρετικό πρόθεμα `@repo` περιορίζει τον κανόνα σε ένα μόνο αποθετήριο. Για
  ένα διαθέσιμο πακέτο αυτό είναι το υποψήφιο repo του· για ένα εγκατεστημένο πακέτο
  είναι το repo από το οποίο ήρθε το πακέτο.

Επειδή το regex ταιριάζεται έναντι όλου του id και είναι χωρίς άγκυρα:

- ένα σκέτο `vlc` ταιριάζει το `vlc` αλλά και το `vlc-plugin-*` (substring),
- το `^vlc-[0-9]` ταιριάζει μόνο το πακέτο `vlc`,
- το `xf86-.*-202.*` ταιριάζει builds με ημερομηνία-έκδοση όπως
  `xf86-video-intel-20260518_...`,
- αγκύρωσε με `^` και `$` για αυστηρό ταίριασμα.

Το επίσημο αποθετήριο είναι απλώς ένα repo με ένα όνομα. Χρησιμοποίησε αυτό το όνομα
με `@`. Σε προεπιλεγμένη ρύθμιση είναι `slackware`, οπότε οι κανόνες μόνο-για-επίσημο
μοιάζουν με `@slackware PATTERN`.

## Μέθοδος 1: η εντολή `frozen`

```
slacker frozen RULE [RULE ...]
```

Κάθε όρισμα είναι ένας κανόνας. Η εντολή επικυρώνει κάθε κανόνα, δείχνει τι θα
παγώσει ο καθένας, ζητά επιβεβαίωση, και μετά προσαρτά τους νέους κανόνες στο
`/etc/slacker/blacklist`. Απαιτεί root.

### Εισαγωγικά (quoting)

Τα εισαγωγικά είναι για το shell, όχι για το slacker. Χρησιμοποίησε διπλά εισαγωγικά
όταν ένας κανόνας:

- περιέχει κενό (οποιοσδήποτε κανόνας `@repo PATTERN`), ή
- περιέχει shell glob ή μεταχαρακτήρα: `*` `?` `[` `]` `(` `)` `|` `$`.

Τα σκέτα ονόματα και το `series/` δεν χρειάζονται εισαγωγικά. Όταν αμφιβάλλεις, βάλε
εισαγωγικά· τα διπλά εισαγωγικά δεν βλάπτουν ποτέ.

```
slacker frozen vlc                 # εντάξει χωρίς εισαγωγικά
slacker frozen kde/                # εντάξει χωρίς εισαγωγικά
slacker frozen "xlibre-*"          # ΥΠΟΧΡΕΩΤΙΚΑ: το * είναι shell glob
slacker frozen "@alienbob vlc"     # ΥΠΟΧΡΕΩΤΙΚΑ: περιέχει κενό
slacker frozen "@alienbob vlc-*"   # ΥΠΟΧΡΕΩΤΙΚΑ: κενό (@repo) ΚΑΙ glob *
```

Τι κάνει στην πράξη το `"@alienbob vlc-*"`: το μέρος `@alienbob` περιορίζει τον
κανόνα στο repo `alienbob`, και το `vlc-*` ταιριάζεται ως **unanchored regex**
έναντι του πλήρους id — και σε ένα regex το `-*` σημαίνει «μηδέν ή περισσότερες
παύλες», όχι «οτιδήποτε». Έτσι παγώνει κάθε εγκατεστημένο πακέτο `alienbob` του
οποίου το id **περιέχει** `vlc` (π.χ. `vlc`, `vlc-plugin-qt`), ακριβώς όπως θα
έκανε ένα σκέτο `vlc`· το `-*` δεν προσθέτει τίποτα. Για να παγώσεις μόνο το πακέτο
`vlc`, αγκύρωσέ το και χρησιμοποίησε character class: `"@alienbob ^vlc-[0-9]"`. (Για
να ταιριάξεις «vlc ακολουθούμενο από οτιδήποτε», το regex είναι `vlc-.*`, όχι
`vlc-*`.)

### Ένας κανόνας ανά κλήση ή πολλοί

```
slacker frozen vlc
slacker frozen vlc "xf86-.*-202.*" "kde/" "@alienbob discover"
```

### Μορφές

```
# σχεδόν-ακριβές μεμονωμένο πακέτο (σημείωση: ταιριάζει και ονόματα που το περιέχουν)
slacker frozen mozilla-firefox

# αυστηρά μόνο ένα πακέτο
slacker frozen "^mozilla-firefox-[0-9]"

# regex σε όλα τα repos
slacker frozen "xf86-.*-202.*"
slacker frozen "xlibre-.*"

# μια ολόκληρη σειρά (όλα τα repos)
slacker frozen kde/

# περιορισμός σε ένα repo: ένα πακέτο από συγκεκριμένο repo
slacker frozen "@alienbob discover"

# περιορισμός σε ένα repo: μια ολόκληρη σειρά από συγκεκριμένο repo
slacker frozen "@conraid kde/"

# μόνο επίσημο repo (προεπιλεγμένο όνομα slackware)
slacker frozen "@slackware mozilla-firefox"
slacker frozen "@slackware kde/"
slacker frozen "@slackware ^mozilla-firefox-"
```

### Επικύρωση και προειδοποιήσεις

- Ένα συντακτικό σφάλμα (για παράδειγμα `@repo` χωρίς pattern) είναι μοιραίο: δεν
  γράφεται τίποτα, και όλα τα προβλήματα που βρέθηκαν παρατίθενται μαζί ώστε να
  διορθωθούν με ένα πέρασμα.
- Ένας κανόνας που αναλύεται σωστά αλλά μοιάζει με λάθος επισημαίνεται, και σε
  ρωτάει αν θες να τον δηλώσεις ούτως ή άλλως:
  - ένα `@repo` που δεν ονομάζει κανένα ενεργό αποθετήριο (πιθανό τυπογραφικό),
  - ένα σκέτο regex που περιέχει κενό· τα id πακέτων δεν περιέχουν ποτέ κενά, οπότε
    αυτό συνήθως σημαίνει ξεχασμένο `@` (για παράδειγμα `conraid foo` αντί για
    `@conraid foo`) ή λάθος στα εισαγωγικά.
- Κανόνες που υπάρχουν ήδη στο αρχείο παραλείπονται, και ο αριθμός επιβεβαίωσης
  αντικατοπτρίζει μόνο τους κανόνες που πραγματικά προστίθενται.

### Flags

- `--yes` (ή `-y`): παράκαμψη των προειδοποιήσεων και του prompt επιβεβαίωσης.
- `--config-dir DIR`: χρήση blacklist κάτω από το `DIR` αντί για `/etc/slacker`.

## Μέθοδος 2: επεξεργασία του αρχείου με το χέρι

Η blacklist είναι ένα απλό αρχείο κειμένου στο:

```
/etc/slacker/blacklist
```

Οι κανόνες είναι ένας ανά γραμμή. Οι κενές γραμμές αγνοούνται. Ένα `#` ξεκινά σχόλιο
(ολόκληρη γραμμή ή στο τέλος). Επεξεργάσου το με οποιονδήποτε editor· δεν απαιτείται
καμία εντολή.

Παράδειγμα αρχείου:

```
# Πάγωσε το εγκατεστημένο firefox· κράτα την τρέχουσα build μας.
mozilla-firefox

# Καρφίτσωσε κάθε πακέτο xlibre, οποιαδήποτε έκδοση.
xlibre-.*

# Κράτα πίσω ολόκληρη τη σειρά KDE.
kde/

# Μόνο η build του discover από τον alienbob.
@alienbob discover

# Μόνο επίσημο repo: μην αφήσεις ποτέ το επίσημο δέντρο να τα αντικαταστήσει.
@slackware ^mozilla-firefox-
@slackware kde/
```

Η εντολή `frozen` γράφει ακριβώς τις ίδιες γραμμές, οπότε οι δύο μέθοδοι είναι
εναλλάξιμες. Η επεξεργασία με το χέρι είναι ο τρόπος να αφαιρέσεις ή να αλλάξεις
έναν κανόνα.

## Κατάλογος κανόνων

| Κανόνας που γράφεις            | Τι παγώνει / κρύβει                                                  |
|--------------------------------|---------------------------------------------------------------------|
| `vlc`                          | κάθε πακέτο του οποίου το id περιέχει `vlc` (vlc και vlc-plugin-*)   |
| `^vlc-[0-9]`                   | μόνο το πακέτο `vlc`                                                 |
| `mozilla-firefox`              | κάθε id που περιέχει `mozilla-firefox`                               |
| `xf86-.*-202.*`                | πακέτα xf86 με έκδοση `202x`                                         |
| `xlibre-.*`                    | κάθε πακέτο `xlibre-*`                                               |
| `kde/`                         | ολόκληρη τη σειρά `kde`, όλα τα repos                                |
| `ap/`                          | ολόκληρη τη σειρά `ap`, όλα τα repos                                 |
| `@alienbob vlc`                | `vlc` μόνο όταν προέρχεται από το repo `alienbob`                    |
| `@conraid kde/`                | τη σειρά `kde` μόνο από το repo `conraid`                            |
| `@slackware mozilla-firefox`   | `mozilla-firefox` μόνο από το επίσημο repo                           |
| `@slackware kde/`              | τη σειρά `kde` μόνο από το επίσημο repo                              |

## Παράθεση και αφαίρεση

- Δεν υπάρχει ξεχωριστή εντολή "list"· διάβασε το αρχείο απευθείας:

  ```
  cat /etc/slacker/blacklist
  ```

  Μπορείς επίσης να επιβεβαιώσεις ένα αποτέλεσμα: ένα blacklisted αλλά μη
  εγκατεστημένο πακέτο εξακολουθεί να εμφανίζεται στο `slacker search NAME` και
  `slacker info NAME` σημειωμένο `[blacklisted]`.

- Δεν υπάρχει εντολή "unfreeze". Για να αφαιρέσεις ή να αλλάξεις έναν κανόνα,
  επεξεργάσου το `/etc/slacker/blacklist` και διάγραψε ή τροποποίησε τη γραμμή.

## Σημειώσεις

- Το `frozen` μόνο προσαρτά· δεν αφαιρεί ποτέ. Η εκ νέου προσθήκη ενός υπάρχοντος
  κανόνα δεν κάνει τίποτα (αναφέρεται ως ήδη παρών).
- Το regex ταιριάζεται έναντι του πλήρους id `name-version-arch-build`, όχι μόνο του
  ονόματος. Χρησιμοποίησε άγκυρες τύπου `^name-` όταν χρειάζεσαι ακρίβεια.
- Ο περιορισμός `@repo` χρησιμοποιεί το όνομα του αποθετηρίου από το αρχείο `repos`
  σου. Έλεγξε τα ονόματα με τη λίστα repo που δείχνει το `frozen` όταν ένας κανόνας
  ονομάζει άγνωστο repo, ή διάβασε το `/etc/slacker/repos`.
- Η εγγραφή της blacklist απαιτεί root, επειδή το αρχείο βρίσκεται κάτω από το
  `/etc/slacker`.
