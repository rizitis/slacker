────────────────────────────────────────────────────────────────────────
LOCAL-MIRROR upgrade-dist  (DISTRO_UPGRADE_MIRROR / distro-upgrade.conf)
────────────────────────────────────────────────────────────────────────
upgrade-dist can now pull the target release from a LOCAL source instead of
re-pointing the remote mirror — a local http/file mirror, an NFS/disk clone,
or a mounted install ISO.

NEW config file: /etc/slacker/distro-upgrade.conf
  Single optional key: DISTRO_UPGRADE_MIRROR
  Absent/empty  -> normal remote dist-upgrade (segment rewrite), unchanged.
  Set           -> upgrade-dist uses that local mirror as the source.

Two shapes supported:
  (1) Mirror tree WITH a slackware{64}-<release> segment in the path:
      slacker reads the release from the path and REQUIRES it == target
      (strong guard against the wrong release).
        e.g. http://nas/slackware/slackware64-current/
             file:///srv/slackware64-current/
  (2) Mounted ISO / path with NO release segment:
      release can't be read from the path -> warn + proceed on the explicit
      target argument + typed confirmation.
        e.g. file:///mnt/iso/

Validation runs BEFORE the point of no return:
  * release check (strict for shape 1, warn for shape 2)
  * PACKAGES.TXT must be reachable at <mirror>/PACKAGES.TXT, else STOP.
Nothing is touched on failure.

Transform: instead of the 15.0->target segment rewrite, the active `mirrors`
line is set to the local mirror (all other active lines commented; idempotent).
`mirror`/`mirror/patches` repos follow it; third-party repos still disabled.

End-of-dist: if a local mirror was used, slacker reminds you it now points at
the local source and to set a remote -current mirror (mirrors / find-mirror)
and clear DISTRO_UPGRADE_MIRROR, so future `slacker update` works online.

Why this is safe in one shot: slacker's TLS is compiled in (bundled CA roots)
and file:// is read directly — NO system libssl/glibc dependency, so it keeps
running while glibc/openssl are replaced underneath it.

Files changed: config.rs, main.rs.  New: conf/distro-upgrade.conf.
Build: 0 warnings, 147 tests (was 144; +distro_upgrade_mirror_parse,
+set_active_mirror_line x2).

BATCH (live-testing follow-ups)  —  config.rs unchanged; main.rs only
────────────────────────────────────────────────────────────────────────
(a) new-config now uses slackpkg's K/O/R/P scheme:
      (K)eep current — .new files LEFT in place for later
      (O)verwrite all — old saved as .orig
      (R)emove all .new
      (P)rompt one by one   [default P]
    Per-file (O)verwrite also saves the old file as .orig.
    Fixes the old bulk [R]eview vs per-file [R]emove letter clash.

(b) Choice prompts now show the KEY letters in WHITE (ui::white) so they
    stand out from the blue prompt text — new-config (bulk + per-file) and
    the [y/N] of confirm(). New helper choice_line().

(c) Release-mismatch guard (HARD REFUSE): install/upgrade now refuses to
    install a package from a repo built for a different Slackware release
    than the running system (e.g. an alienbob -current repo on 15.0).
    GPG/md5 authenticate WHO built a package, not WHICH RELEASE.
      - new helper repo_release_token(url): detects release from both
        slackware{arch}-<suffix> AND a bare current / X.Y path segment
        (catches alienbob sbrepos/current); None for release-agnostic repos.
      - execute_plan refuses on mismatch; --yes overrides.
      - upgrade-dist is unaffected (separate code path).
      - status gained a "Repo release" row warning of any mismatch.

(d) upgrade-dist: the in-dist update phase no longer shows the per-repo
    menu. After the typed point-of-no-return it prints one line and a single
    confirm: "Connected to the -current repositories. Proceed with the
    distribution upgrade? [Y/n]" (default Y), then updates all repos silently.

(e) upgrade-dist: the GnuPG verification chain (gnupg/gnupg2/gpgme/
    libassuan/libgpg-error/npth — DIST_GPG_LAST) is now upgraded LAST, so the
    working gpg keeps verifying every other package's signature throughout the
    run instead of falling back to md5 mid-stream.

(f) SIGPIPE: `slacker ... | head` no longer panics with "failed printing to
    stdout"; SIGPIPE is reset to its default disposition at startup (raw libc
    signal call, no new crate dependency).

Build: 0 warnings, 150 tests (was 147; +repo_release_token / release_version
/ dist_defers_gpg). Files changed: main.rs only.
